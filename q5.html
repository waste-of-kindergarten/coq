Apply 策略

先用一个例子做一个比较直观的认识。

Theorem simp_ex1 : forall n m : nat ,
 (n,n) = (m,m) -> 
 (forall q r : nat , 
 (q,q) = (r,r) -> [q] = [r]) 
 -> [n] = [m].
Proof.
intros n m eq1 eq2.
apply eq2.
apply eq1.
Qed.

在Coq中可以逐步观察推到过程，apply有些类似rewrite + reflexivity的功能，相比更为强大，示例中apply eq2时，Coq尝试将eq2中的结论与当前目标进行匹配，eq2的结论中[q] = [r](注意任意性)可以和[n] = [m]进行匹配，从而将目标改写为(n,n) = (m,m),这与eq1相匹配(可以看作真->(n,n)=(m,m)),于是完成了目标的证明。

我个人理解的证明过程：

  • 要证 [n] = [m], 只需证 (n,n) = (m,m)
  • 而(n,n) = (m,m) 即为假设条件
  • 证毕

apply策略虽然强大，但是匹配机制要求精确匹配，如果出现等式颠倒apply将失效，此时应用symmetry策略可以解决这一问题。

    Theorem simp_ex2 : forall a b:nat,
    a = b -> b = a.
    Proof.
    intros a b H.
    symmetry.
    apply H.
    Qed.

思考：apply 和 rewrite 策略之间有什么区别。我们将在下面给出一个示例用来说明这一问题。

apply with 策略

等式的传递性，是一种十分常见的模式，可以作为一条引理记录下来。

Theorem trans_eq : forall (X:Type) (n m o : X), 
n = m  -> m = o -> n = o.
Proof.
intros X n m o eq1 eq2.
rewrite eq1.
rewrite eq2.
reflexivity.
Qed.

对于某些问题需要证明实例的等式传递性，我们希望用到上面的引理。

Theorem trans_eq_example1:
 forall a b c d e f : nat,
[a;b] = [c;d] -> 
[c;d] = [e;f] -> 
[a;b] = [e;f].

根据apply的匹配机制，能够apply trans_eq.能匹配到n := [a;b] 和 o := [e;f],但是没有匹配到m.

使用apply with策略可以显式进行匹配。如下：

Proof.
intros a b c d e f.
apply trans_eq with (m:=[c;d]). 
apply eq1.
apply eq2.
Qed.

回顾

思考：apply 和 rewrite 策略之间有什么区别。我们将在下面给出一个示例用来说明这一问题。
trans_eq_example1使用rewrite策略 和 trans_eq引理 证明

rewrite trans_eq with 
(n:=[a;b]) (m:=[c;d]) (o:=[e;f]).
reflexivity.
apply eq1. apply eq2. Qed.

尝试使用 rewrite trans_eq.给出的报错时Unable to find an instance for the variables m, o.也就是说此时只匹配了n,从这里可以看出rewrite 匹配机制和apply的不同。

injection 和 discriminate 策略

定义递归数据类型时，实际上满足了两个条件：

  • 所有构造子时单射的或一一对应的
  • 不同的构造子之间不能相交

因此根据这两条事实，衍生injection和discriminate策略。

对于injection策略，即脱去构造子不影响等式成立。例如典型的自然数构造：

Theorem S_injective : forall n m : nat,
S n = S m -> n = m.
Proof.
intros n m H.
injection H as Hnm.
apply Hnm.
Qed.

对于discriminate策略，即不同构造子的等式一定不成立，例如自然数的构造子S和O。

矛盾的前提推出任何东西都是真命题：

Theorem discriminate_ex : forall n : nat ,
 S n = 0 -> 1 + 1 = 3.
Proof.
intros n contradition.
discriminate contradiction.
Qed.

unfold 展开定义

某些情况下手动展开定义是必要的，Coq提供了unfold Definition来对目标进行展开。

《software foundation》中给出的例子是：

Definition square n := n * n.

Lemma square_mult : forall n m , square (n * m) = (square n) * (square m),
Proof.
intros n m.
unfold square. 
(* 使用 simpl. 没有任何改变 *)
rewrite mult_assoc.
assert (H:n * m * n = n * n * m).
{rewrite mult_comm. apply mult_assoc. }
rewrite H.
rewrite mult_assoc.
reflexivity.
Qed.

simpl 和 reflexivity 以及 apply 均能够一定程度上自动展开定义，但是这种自动展开具有一定的“保守”性质。

我们举一个最早学到的自然数的例子:

Example pred_simpl:forall x:nat, pred (S x) = x.
Proof.
intros x.
simpl.
reflexivity.
Qed.

Example pred_unfold : forall x:nat , pred (S x) = x.
Proof.
intros x.
unfold pred.
relflexivity.
Qed.

这个例子中既能够使用自动展开，也能够使用手动展开，下面展示一个不能自动展开的例子：

Example plus_unfold : forall x : nat, plus 0 x = x.
Proof.
intros x.
unfold plus.
destruct x as [|x'].
- reflexivity.
- rewrite plus_comm.
reflexivity.
Qed.

Example plus_simpl: forall x : nat, plus x 0 = x.
Proof.
intros x.
(*此处 simpl 没有任何进展*)
destruct x as [|x'].
- simpl. reflexivity.
- simpl. rewrite plus_comm. reflexivity.
Qed.

对于上面的例子，如果将x置为常量，则可以直接使用simpl.

在第一个例子中，根据pred的定义，有两种模式匹配:O和S _ ,示例中要证明等式的左侧pred的参数明显符合S _的模式，因而可以自动展开，当然也可以手动展开。

第二个例子，由于plus的定义中需要匹配x的模式，一种是O,另一种是S _,对于simpl来说这将难以进行下去，于是放弃展开。

其实只要没有明确的递归定义就无法使用simpl，为此我们回到square定义中来。我们使用模式匹配进行几种等价的定义:

Definition square (n:nat) : nat :=
match n with
| _ => n * n
end.

Definition square (n : nat): nat :=
match n with
| 0 => 0
| _ => n * n
end.

Example square_test : forall x : nat , square (S x) = (S x) * (S x).

使用第一个定义时发现使用simpl没有任何变化，与Definition suqare n := n * n.效果相同，而第二个定义却可以将square展开.我们看到第一个定义提供了一个假的模式匹配，但是仍然无法展开，因此simpl展开生效的条件需要变量是确定的递归模式。

对复合表达式使用 destruct

destruct 除了可以对变量进行分情况讨论，也可以对表达式进行分情况讨论。

Definition fun (n: nat) : bool :=
    if n =? 3 then false
    else if n =? 5 then false
    else true.

Theorem fun_exa : forall n : nat,fun n = false.
Proof.
intros n. unfold fun.
deestruct (n =? 3) eqn : E1.
    - reflexivity.
    - destruct n =? 5 eqn : E2.
        + reflexivity.
        + reflexivity.
Qed.
