归纳法则

基础

每次使用 Indutive 来声明数据类型时，Coq会自动为该类型生成'归纳法则'。这些法则名称是'数据类型_ind'，例如nat_ind,list_ind等。

通常，为归纳类型t生成的归纳法则形式如下:

  • 对每个构造子c生成归纳法则中的一种情况
  • 如果c不接受参数，该情况为
    "P 对 c 成立"
  • 如果c接受参数 x1:a1 ... xn:an,该情况为:
    "对所有的x1:a1 ... xn:an,如果P对每个类型为t的参数都成立，则P对c x1 ... xn 成立"

示例：

Inductive list (X:Type) : Type :=
| nil
| cons (n : X) (l : natlist).
Check list_ind.
(*list_ind
     : forall (X : Type)
         (P : list X -> Prop),
       P (nil X) ->
       (forall (n : X) (l : list X),
        P l -> P (cons X n l)) ->
       forall l : list X, P l*)

Inductive season : Type :=
| spring
| summer
| autumn
| winter.
Check season_ind.
(*season_ind
     : forall P : season -> Prop,
       P spring ->
       P summer ->
       P autumn ->
       P winter ->
       forall s : season, P s*)

可以看到，无论是否为递归的，Coq都会生成对应的归纳法则，归纳原理仍然可用来证明其性质。

Theorem n_plus_one_is_S_n : forall n : nat,
n + 1 = S n.
Proof.
apply nat_ind.
-reflexivity.
-intros n IHn.
simpl. rewrite IHn.reflexivity.
Qed.

Theorem n_plus_one_is_S_n1 : forall n : nat,
n + 1 = S n.
Proof.
intros n.
induction n.
- reflexivity.
- simpl. rewrite IHn. reflexivity.
Qed.

induction的工作就是应用归纳类型对应的归纳法则,只不过增加了变量名的管理工作，在实践中使用induction更加方便。

深入induction 策略

考虑自然数的数学归纳法（非形式化陈述）:
如果P n是某个涉及数字n的命题，我们想要证明P对于所有数字n都成立，使用如下方式推理：

  • 证明P 0 成立
  • 证明P n'成立，那么P (S n')成立
  • 得出结论P n对所有n成立

当我们使用intros n 和 induction n 证明时，Coq 实际上先考虑了一个特殊的n(引入上下文),然后通过归纳告诉它证明一些关于全体数字的性质。此时，Coq再次一般化了进行归纳的变量。

当然也可以不对n进行特殊化——对于目标中含有量词的变量应用induction也可以工作。

Theorem plus_assoc' : forall n m p : nat, n + (m + p) = (n + m) + p.
Proof.
intros n m p.
induction n as [n' IHn'].
(*重新一般化*)
- reflexivity.
- simpl. rewrite -> IHn'. reflexivity. Qed.

Theorem plus_assoc'' : forall n m p : nat, n + (m + p) = (n + m) + p.
Proof.
(*不进行特殊化*)
induction n.
- simpl. reflexivity.
-intros m p.
simpl. rewrite IHn. reflexivity.
Qed.

Thorem plus_assoc''' : forall n m p : nat , n + (m + p) = (n + m) + p.
Proof.
intros n m p.
generalize n as n0.
(*将n取消特殊化*)
induction n0.
- reflexivity.
- simpl. rewrite IHn0. reflexivity.
Qed.

Prop 中的归纳法则

考虑之前学习的ev定义。

Inductive ev : nat -> Prop :=
| ev_0 : ev 0
| ev_SS : forall n : nat ,ev n -> ev (S (S n)).
Check ev_ind.
(*ev_ind
     : forall P : nat -> Prop,
       P 0 ->
       (forall n : nat,
        ev n -> P n -> P (S (S n))) ->
       forall n : nat, ev n -> P n*)

按照之前的归纳法则，ev_ind似乎应该是这样的：

ev_ind' : forall P :( forall n : nat , ev n -> Prop),
P 0 ev_0 -> 
(forall (m : nat) (E:ev m),
P m E -> P (S (S m))) (ev_SS m E)) ->
forall (n:nat) (E:ev n), P n E.

然而我们真正想要的是证明某些自然数是偶数这样的性质，即关于自然数的断言而非证据，如果对于命题P的归纳法仅仅被n所参数化，且其结构是P对所有偶数n成立，那么会方便许多。因此Coq为ev生成了简化的归纳法则。

该归纳法则用自然语言表述：

假设P是关于自然数的一个性质,为证明当n是偶数时P n成立，需要证明:

  • P 对 0 成立
  • 对任意n,如果n是偶数且P对n成立，那么P对S (S n)成立。

另外，Inductive定义的具体形式会影响到Coq生成的归纳法则。

例如

Inductive le1 : nat -> nat -> Prop :=
| le1_n : forall n , le1 n n
| le1_S : forall n m , (le1 n m) -> (le1 n (S m)).

Inductive le2 (n : nat) : nat -> Prop :=
| le2_n : le2 n n 
| le2_S m (H : le2 n m) : le2 n (S m).

Check le1_ind.

(*le1_ind
     : forall P : nat -> nat -> Prop,
       (forall n : nat, P n n) ->
       (forall n m : nat,
        le1 n m -> P n m -> P n (S m)) ->
       forall n n0 : nat,
       le1 n n0 -> P n n0*)

Check le2_ind.

(*le2_ind
     : forall (n : nat)
         (P : nat -> Prop),
       P n ->
       (forall m : nat,
        le2 n m -> P m -> P (S m)) ->
       forall n0 : nat,
       le2 n n0 -> P n0*)

第二个生成了更简单的归纳法则。

总结

《软件基础》给出了两个关于归纳的模板，这对于学习归纳的底层原理是很有帮助的。

对归纳定义的集合进行归纳

有形如"forall n : S ,P (n)"的全称量化命题，其中S是某个归纳定义的集合。对n进行归纳。

对S中的每个构造子c的情形:

  • 假设n = c a1 ... ak , 为每个具有类型S的a陈述归纳假设IH,我们需要证明P(c a1 ... ak)成立。(继续证明至完成此情形)
  • 其他情形以此类推

对归纳定义的命题进行归纳

有形如"Q -> P"的命题，其中Q是某个个归纳定义的命题。证明对Q的导出式进行归纳。

对Q中的每个构造子c的情形:

  • 假设被用于证明Q的最终规则是c，在这里，我们陈述了所有a类型，以及构造子定义的所有等式，以及所有类型为Q的a的IH(如果有的话)
