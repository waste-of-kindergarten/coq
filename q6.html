LOGIC COQ中的逻辑系统

逻辑联结词

合取

命题A与B合取写作"A /\ B"，表示一个A与B均为真的断言。

合取的命题证明

证明合取的命题，即分别证明每个被合取的命题都是真的。

Example and_example : forall A B: Prop , A -> B -> A /\ B.
Proof.
intros.
split.
apply H. apply H0.
Qed.

这里面出现了一个新的策略split,它可以将合取的命题进行拆分，形成若干个子目标。

带有合取前提的命题证明

考虑两个关于合取的蕴含结论P /\ Q -> P,P /\ Q -> Q.

Lemma proj1 : forall P Q : Prop,
P /\ Q -> P.
Proof.
intros P Q HPQ.
destruct HPQ as [HP _].
apply HP. Qed.

Lemma proj2 : forall P Q : Prop,
P /\ Q -> Q.
Proof.
intros P Q HPQ.
destruct HPQ as [_ HQ].
apply HQ. Qed.

destruct 可以将前提中的合取式进行解构，形成两个前提.[HP _]以及[_ HQ]将结构的前提进行命名，_是占位符，即表示该前提存在但是不为其分配名称（因为不会使用该前提）。

除了destruct 还可以更早地进行隐式分解，见下面结论律的证明。

顺便列举一些经常使用到的合取结论。

交换律

Theorem and_comm : forall P Q : Prop,
P /\ Q -> Q /\ P.
Proof.
intros P Q [HP HQ].
split.
- apply HQ.
- apply HP.
Qed.

结合律

Theorem and_assoc : forall P Q R : Prop,
P /\ (Q /\ R) -> (P /\ Q) /\ R.
Proof.
intros P Q R [HP [HQ HR]].
(*隐式拆分*)
split.
- split. apply HP. apply HQ.
- apply HR.
Qed.

注意: 交换律和结合律都是双向的，这里只证明单向是因为<->还没有学习。

析取

析取的命题证明

对于析取的命题A \/ B,只需A或B二者之一为真即可.因此在证明时需要将析取的命题拆分成A或B。

Lemma or_impl1 : forall A B: Prop , A -> A \/ B.
Proof.
intros A B HA.
left.
apply HA.
Qed.  

Lemma or_impl2 : forall A B :
Prop , B -> A \/ B.
Proof.
intros A B HB.
right.
apply HB.
Qed.

利用left 和 right策略可以选择析取命题的的左侧和右侧作为新的目标。

带有析取前提的命题证明

当前提是析取命题时，必须保证被析取的任意命题为真的情况下，结论为真。因而，需要将前提拆分成若干情况分别证明。

交换律

Theorem or_comm : forall A B : Prop,
A \/ B -> B \/ A.
Proof.
intros A B H.
destruct H as [HA | HB].
(*显式拆分*)
- right. apply HA.
- left. apply HB.
Qed.

结合律

Theorem or_assoc : forall P Q R : Prop,
P \/ (Q \/ R) -> (P \/ Q) \/ R.
Proof.
intros P Q R [HP | [HQ | HR]].
(*隐式拆分*)
- left. left. apply HP.
- left. right. apply HQ.
- right. apply HR.
Qed.

假命题与否定

在Coq中，not定义如下:

Definition not (P : Prop) := P -> False.
Notation "~ x" := (not x) : type_scope.

自反律

Theorem double_neg : forall P : Prop,
P -> ~ ~ P.
Proof.
intros P HP.
unfold not.
intros H1.
(*引入新的前提*)
apply H1. apply HP.
Qed.

矛盾律

Theorem contradiction : forall P : Prop, (P /\ ~ P) -> True.
Proof.
intros P [HP HNP].
unfold not in HNP.
apply HNP in HP.
destruct HP.
(*爆炸原理*)
Qed.

False本身是个矛盾性的命题，如果让False进入到证明的上下文，那么可以对其使用destruct 来完成证明。即爆炸原理'从谬误出发，可以证明任何命题'。

逻辑等价

逻辑等价，即“当且仅当”，定义如下。

Definition iff (P Q : Prop) := (P -> Q) /\ (Q -> P).
Notation "P <-> Q" := (iff P Q)(at level 95,no associativity): type_scope.

根据定义，我们当然可以在证明中将其视为合取的命题，利用split策略分别证明。

对称性

Theorem iff_sym : forall P Q : Prop, (P <-> Q) -> (Q <-> P).
Proof.
intros P Q [HPQ HQP].
split.
(*先证 P -> Q*)
- apply HQP.
(*再证 Q -> P*)
- apply HPQ. 
Qed.

分配律

Theorem or_distributes_over_and : forall P Q R : Prop,
P \/ (Q /\ R) <-> (P \/ Q) /\ (P \/ R).
Proof.
intros P Q R.
split.
+intros [HP | [HQ HR]].
(*两种情况讨论*)
-split. left. apply HP. left. apply HP.
-split.  right. apply HQ. right. apply HR.
+intros [[HP | HQ]  [HP1 | HR]].
(*四种情况讨论*)
- left. apply HP.
-left. apply HP.
- left. apply HP1.
- right. split. apply HQ. apply HR.
Qed. 

Theorem and_distrubutes_over_or:
forall P Q R: Prop,
P /\ (Q \/ R) <-> (P /\ Q) \/ (Q /\ R).
Proof.
intros P Q R.
split.
+ intros [HP [HQ | HR]].
(*两种情况讨论*)
-left. split. apply HP. apply HQ.
-right. split. apply HP. apply HR.
+ intros [[HP HQ] | [HP1 HR]].
(*两种情况讨论*)
- split. apply HP. left. apply HQ.
-split. apply HP1. right. apply HR.
Qed.

广集与逻辑等价

Coq的某些策略hi特殊对待iff语句，从而避免某些底层的证明状态。特别地，rewrite 和 reflexivity 也可用于iff语句。

From Coq Require Import Setoids.Setoid.

在Coq 8.15版本下，似乎无需导入即可使用rewrite和reflexivity 对等价进行操作。

”广集“指配备了等价关系的集合，即满足自反性、对称性和传递性的关系。当一个集合中的两个元素在这种关系等价时，可以用rewrite 将其中一个元素替换为另一个。

例如：

Lemma mult_0 : forall n m , n * m = 0 <-> n = 0 \/ m = 0.
Admitted. (*证明略*)
Lemma mult_0_3 :
forall n m p, n * m * p = 0 <->n = 0 \/ m = 0 \/ p = 0.
Proof.
intros n m p.
rewrite mult_0.
rewrite mult_0.
rewrite or_assoc.
reflexivity.
Qed.

除了rewrite 和 reflexivity外，apply策略也可以用在<->上，Coq将尝试猜出正确的方向。

例如将mult_0的一个方向拆出来作为命题。

Lemma apply_iff_example:
forall n m : nat , n * m = 0 -> n = 0 \/ m = 0.
Proof.
intros n m H.
apply mult_0.
apply H.
Qed.

存在量词

我们最早接触的是全称量词forall,现在介绍存在量词exists.

先看一个示例:

Lemma exist_example : exists n : nat, S n = 1.
Proof.
exists 0.
reflexivity.
Qed.

为了证明形如'exists x,P'的语句，需要找到合适的值替代x,然后证明替换后的命题。exists 0将n替换成0.

如果上下文中有形如exists x,P的存在前提，可以将其解构。

Example exist_example1 : (exists n :nat ,S n = 1) -> (exists m, S m = 1).
Proof.
intros [n Hn].
exists n.
apply Hn.
Qed.

尝试将exists n放到大前提中，可以发现，此时整个命题变为exists x,P，此时需要对x赋值才能继续下去。

否定对量词的作用

Theorem forall_not_exists : forall (X:Type) (P: X -> Prop),
(forall x, P x) -> ~ (exists x, ~ P x).
Proof.
unfold not.
intros X P H [x Px].
apply Px.
apply H.
Qed.

对参数使用定理

Coq的特性是将证明本身看作一等对象——函数，所以我们可以像使用函数一样使用定理。

例如，我们做如下检查：

Check plus_comm.
(*plus_comm
     : forall n m : nat,
       n + m = m + n*)
Check plus_comm 1.
(*plus_comm 1
     : forall m : nat,
       1 + m = m + 1*)
Check plus_comm 1 2.
(*plus_comm 1 2
     : 1 + 2 = 2 + 1*)
Check plus_comm _ 1.
(*plus_comm ?n 2
     : ?n + 2 = 2 + ?n
where
?n : [ |- nat]*)
Check plus_comm : forall a b :nat, a + b = b + a.
(*plus_comm
:
forall a b : nat, a + b = b + a
     : forall a b : nat,
       a + b = b + a*)
Example plus_copy : forall p q : nat,
p + q = q + p -> 1 + 2 = 2 + 1.
Admitted.
Check plus_copy _ _ plus_copy.

由于定理可以作为函数，被应用刀对应类型的前提上，因此可以在证明中直接产生结论，而不是使用断言，

下面对一个比较复杂的示例进行分析，以便读者能够完全理解这种用法。

(*几个前提*)
Fixpoint In {A : Type} (x : A) (l : list A) :Prop :=
match l with
| [] => False
| x' :: l' => x'= x \/ In x l'
end.


Theorem in_not_nil:
forall A (x: A) (l: list A) , In x l -> l <> [].
Proof.
intros A x l H. unfold not.
intros H1. destruct l eqn : HE.
-simpl in H. destruct H.
-discriminate H1.
Qed.

Lemma In_map_iff :
forall (A B : Type) (f : A -> B) (l: list A) (y : B),
In y (map f l) <-> exists x ,f x = y /\ In x l.
Proof.
intros A B f l y. split.
Admitted.

(*一个复杂示例*)
Exmaple lemma_application_ex : 
forall {n : nat} {ns :list nat}，
In n (map (fun m => m * 0) ns) -> n = 0.
Proof.
intros n ns H.
destruct (proj1 _ _ (In_map_iff _ _ _ _ _) H) as [m [Hm _]].
rewrite mult_0_r in Hm. rewrite <- Hm. reflexivity.
Qed.

注意这个位置destruct (proj1 _ _ (In_map_iff _ _ _ _ _) H) as [m [Hm1 Hm2]]..我们来分析destruct 到底拆了什么东西。

首先，定理除了接收特定参数外，还可以在参数后面添加假设一遍应用定理(正推).首先proj1 接收两个参数，这两个参数用_代替,Coq可以自动推理得出参数，在参数后面又接收了一个定理(In_map_iff _ _ _ _ _), In_map_iff是双向的，根据定义可以分解为两个定理合取，这刚好匹配到了proj1的前提. 当proj1 应用到 In_map_iff 后，proj1 本身的参数被In_map_iff 中具体内容替代，此时未知参数变为了In_map_iff 中的五个参数，再次将整个式子应用在H上，H也恰好匹配了前提，从而得到了一个较为复杂的式子。在定理证明时可以利用Check可以对其进行检查。

proj1
  (In n
     (map (fun m : nat => m * 0) ns) ->
   exists x : nat,
     (fun m : nat => m * 0) x = n /\
     In x ns)
  ((exists x : nat,
      (fun m : nat => m * 0) x = n /\
      In x ns) ->
   In n
     (map (fun m : nat => m * 0) ns))
  (In_map_iff nat nat
     (fun m : nat => m * 0) ns n) H
     : exists x : nat,
         x * 0 = n /\ In x ns

destruct 将函数推到结果x * 0 = n /\ In x ns进行拆分，将命名为m,并将合取拆分成两个前提。

m : nat
Hm1 : m * 0 = n
Hm2 : In m ns

注意到整个证明并未用到Hm2这个前提,destruct ... as后面可以使用哑变量_来替代Hm2，此时Hm2将不被引入到前提.
