多态与高阶函数

多态

在编程语言和类型论中，多态指为不同数据类型的实体提供统一的接口。 多态类型可以将自身所支持的操作套用到其它类型的值上。

使用多态列表可以一次性定义布尔列表，自然数列表，列表的列表等等。

    Inductive list (X:Type) : Type :=
        | nil
        | cons (x:X) (l:list X).

这里面list可以视为从Type 类型到Type类型的函数.

    Check list : Type -> Type.
    Check nil : forall X : Type, list X.
    Check cons : forall X : Type , X -> list X -> list X.

每次使用构造子时都需要提供相应的类型，这很麻烦，Coq提供了一定的自动推断机制。

类型标注推断

以repeat为例，按照目前的已知可以定义为：

    Fixpoint repeat (X:Type) (x:X) (count : nat) : list X :=
    match count with
        | 0 => nil X
        | S count' => cons X x (repeat X x count')
        end.

由于上面定义中返回类型为list X,这表明X一定为Type 类型，而后面的cons构造子在确定了X类型后又接收了x作为参数，根据cons的定义x必定为X类型，根据S count'又可推断出count是nat类型。所以似乎没有必要写如此完整。

事实上，Coq确实可以进行这样的推断，如下可以简化为

    Fixpoint repeat' X x count : list X :=
        match count with
            | 0 => nil X
            | S count' => cons X x (repeat' X x count')
            end.

    Check repeat':
    forall X : Type, X -> nat -> list X.
    Check repeat :
    forall X : Type , X ->
    nat -> list X.

类型参数的推断

在Coq中提供了一种“洞”的概念，可以通过_符号让Coq进行自动填补，如下：

    Fixpoint repeat'' X x count : list X :=
        match count with
            | 0 => nil _
            | S count' => cons _ x (repeat'' _ x count')
            end.

上面的定义中，很显然_处都应该补充X。

隐式参数

即使是洞有时也会让人觉得麻烦，在一些高级的语言中，多态的类型通常是隐式指明的，在Coq中也提供了类似机制。Argument用于指定函数或构造子的名字并列出参数名，使用花括号{参数}将参数视为隐式参参数。

    Arguments nil {X}.
    Arguments cons {X} _ _.
    Arguments repeat {X} x count.

注意到cons (x:X) (l:list X)中接受的两个参数没有名字，使用洞来指示Coq进行推断，而repeat (X:Type) (x:X) (count : nat) 接受的参数又名字，可以直接写参数名称（当然也可以使用"洞"）。

从而可以将repeat继续化简为：

    Fixpoint repeat''' {X : Type} (x : X) (count: nat) : list X :=
        match count with
            | 0 => nil
            | S count' => cons x (repeat''' x count')
            end.

注意：简略形式需要同时照顾到操作和阅读两方面，一般不建议直接在定义中隐式声明类型参数，而是在显式定义完多态类型后，再使用Argument进行声明，例如，如果

inductive list' {X:Type} : Type :=
  | nil'
  | cons' (x : X) (l : list').

如是定义，那么repeat 函数的返回类型甚至不能写成list X.这会使得阅读变得更加困难。

折中：显式提供类型参数

已经将参数隐式声明后，仍然可以显式将类型提供给构造子或函数，使用@:

    Definition mynil' := @nil nat.

细心的读者也许发现，刚刚的“注意”中的内容可以写成如下形式：

    Inductive list' {X:Type} : Type :=
        | nil'
        | cons' (x:X) (l : @list X).

    (*repeat 似乎也可以这样写*)
    Fixpoint repeat'''' {X} x count : @list X:=
        match count with
            | 0 => nil
            | S count' => cons x (repeat'''' x count')
            end.

确实可以这样来写，只是在阅读时仍然不如建议的方法更容易理解，《software foundation》作者仍然建议使用规范的方法进行定义。

多态序对

类似的，可以定义多态序对。

    Inductive prod (X Y : Type) : Type :=
        | pair (x:X) (y:Y).
    Arguments pair {X} {Y} _ _.

使用Notation 定义标准的“积类型”：

    Notation "X * Y" := (prod X Y) : type_scope.

type_scope 指令Coq该记法只能在解析类型而非表达式中使用，这样可以避免与乘法符号的冲突。

示例：

    Definition fst {X Y:Type} (p : X * Y) : X :=
        match p with
            | (x, y) => x
            end.

多态候选

推广上一节的natoption定义option版本。

    Inductive option (X:Type) : Type :=
        | Some (x : X)
        | None.

    Arguments Some {X} _.
    Arguments None {X}.

    Fixpoint nth {X: Type} (l : list X) (n : nat): option X :=
        match l with
            | [] => None
            | a :: l' => match (eqb n 0) with
                             | true => Some a
                             | _ => nth l' (pred n)
                             end
            end.

函数作为数据

Coq允许将函数作为参数传入气态函数，作为结果返回以及存储在数据结构中等。

高阶函数

用于操作其他函数的函数通常叫"做高阶函数"

    Fixpoint iteration {X:Type} (f : X -> X) (n:nat) (t:X) : X :=
        match n with
            | 0 => t
            | S n' => (iteration f n' (f t)) 
            end.

示例中iteration定义了一种迭代模式，将函数迭代n次，作用于t上

过滤器

过滤器将函数分别作用于列表中的每个元素,筛选出符合的元素形成新的列表。

    Fixpoint filter {X : Type} (f:X -> bool) (l : list X) : (list X):=
         match l with
            | [] => []
            | h :: t => match (f h) with
                            | true => h :: (filter f t)
                            | _ => filter f t
                            end
            end.
(*注意需要在此之前补全多态列表的记法声明，具体可以参考上一节的记法*)

匿名函数

匿名函数允许在不声明函数名称时使用函数。

    Example test_filter:
    filter (fun n => n >=? 3) [1;2;3;4] = [3;4].

注意：fun是关键字

映射

映射将函数分别作用于列表中的每个元素，作用后的元素形成一个新的列表并返回。

    Fixpoint map {X Y: Type} (f:X->Y) (l:list X) : (list Y ):=
        match l with
            | [] => []
            | h :: t => (f h) :: (map f t)
            end.

折叠

一个更强大的高阶函数是fold,fold操作将一个给定二元函数逐个迭代作用每个元素（一个参数接收上次返回结果，另一个接收列表中下一个元素）

    Fixpoint fold {X Y : Type } (f : X -> Y -> Y) (l:list X) (b: Y) : y :=
        match l with
            | nil => b
            | h :: t => f h (fold f t b)
            end.

    Example fold_test:
    fold mult [1;2;3;4] 1 = 24.
    Proof. reflexivity. Qed.

其中b为起始元素，即"第零次"作用的返回结果。

用函数构造函数

函数作为返回的一个例子

    Definition constfun {X: Type} (x: X) : nat -> X :=
    fun (k:nat) => x.

    Definition ftrue := constfun true.

    Example constfun_example : ftrue 0 = true.
    Proof. reflexivity. Qed.
