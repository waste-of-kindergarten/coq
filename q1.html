数据与函数

枚举类型定义

枚举类型定义了有限的成员组成的数据类型，语法如下：
Inductive typename : Type :=
| member1
| member2
| ... .

示例：定义一个二进制bit类型

	Inductive bit: Type :=
		| b0
		| b1.

示例定义了一个bit类型的数据集合，包含了b0和b1两个成员（代表0和1）.

函数定义

一旦有了数据类型，就可以定义函数，根据bit类型定义取反：

	Definition reverse(b:bit) : bit :=
		match b with
			| b0 => b1
			| b1 => b0
		end.

上面的代码定义了reverse,接收一个bit类型的参数，返回一个bit类型的参数，并定义了 reverse b0 和 reverse b1 分别为 b1 和 b0.

可以利用定义的函数进行计算，关键字是Compute.

	Compute (reverse b0).

Constructor构造器

类型中的所有成员被称为Constructor,而Constructor 可以构建表达式，表达式允许单个Constructor或者多个参数的情况。

下面以色彩类型来说明：

	Inductive rbg: Type :=
		| red
		| green
		| blue.
		
	Inductive color: Type :=
		| black
		| white
		| primary (p: rgb).

模式匹配

正如reverse函数中的match,将 b 分别匹配成b0 和 b1,并给出每种情况的结果，称为模式匹配。

	Definition monochrome (c: color) : bool :=
		match c with
			| black => true
			| white => true
			| primary _ => false
		end.

	Definition isgreen (c:color) : bool :=
		match c with
			| primary green => true
			| _ => false
		end.

第一个函数用来判断颜色实例是否是纯色（白色或者黑色），primary 构造器接受一个参数，所以需要带有一个变量或者常量。所以第一个函数也可以写为：

	Definition monochrome (c: color) : bool :=
		match c with
			| black => true
			| white => true
			| primary p => false
		end.

'_'被称为哑变量，可以理解为匹配任何之前没有匹配的模式。

我们还可以将上述函数再次化简：

	Definition monochrome (c: color) : bool :=
		match c with
			| primary _ => false
			| _ => true
		end.

更复杂的枚举构造 —— 元组方法

利用之前定义的bit类型，我们定义byte类型：

	Inductive byte : Type :=
		| bits (B0 B1 B2 B3 B4 B5 B6 B7 : bit).

不同类型组装成元组，以组装Int类型为例（为了说明方法，定义Int 有4个byte和1个bit 组成）:

	Inductive Int : Type :=
		| integer (sign : bit) (B0 B1 B2 B3: byte).

自然数构造——递归

根据自然数公理，定义自然数如下:

	Inductive N : Type :=
		| O
		| S (n : N).

这里面定义了0元素（用'O'表示），以及任意自然数的后继元素。

后继S 构造器可以看作一个函数

	Check S: N -> N.

相应的可以定义前继函数

	Definition pre (n : N): N :=
		match n with
			| O => O
			| S n' => n'
		end.

递归函数定义

对自然数的奇偶判断就是典型的递归函数：

	Fixpoint is_even (n : N) : bool :=
		match n with
			| O => true
			| S O => false
			| S (S n') => is_even n'
		end.

	Fixpoint is_odd (n : N) : bool :=
		match n with
			| O => true
			| S O => false
			| S (S n') => is_odd n'
		end.

以第一个偶数判断为例，我们知道O是偶数，S O 是奇数，对于任意的数，其与两次前继操作后的元素具有相同的奇偶性，因此任意的元素都可以递归到判断O或S O的奇偶性。

利用递归还可以定义更多的函数

【等于】

	Fixpoint eq (n m : N) : bool :=
		match n with
			| O => match m with
					| O => true
					| _ => false
					end
			| S n' => match m with
						| O => false
						| S m' => eq n' m'
					end
		end.

当然也等价于

	Fixpoint eq (n m : N) : bool :=
		match n,m with
			| O , O => true
			| S n', S m' => eq n' m'
			| _ ,_ => false
		end.

【不等】

	Definition neq (n m : N) : bool := negb (eq n m).

【小于】

	Fixpoint lt (n m : N) : bool :=
		match n,m with
			| O , S m' => true
			| S n' , S m' => lt n' m'
			| _ , _ => false
		end.

【大于】

	Fixpoint gt (n m : N) : bool :=
		match n,m with
			| S n' , O => true
			| S n' , S m' => gt n' m'
			| _ , _ => false
		end.

【大于等于】

	Definition le (n m : N) : bool : = orb (eq n m) (lt n m).

【小于等于】

	Definition ge (n m : N) : bool := orb (eq n m) (gt n m).

当然定义并不是唯一的，大于等于和小于等于也可以用递归来定义。

【加法 减法 乘法 乘方】

Fixpoint plus (n : N) (m : N) : N :=
  match n with
    | O => m
    | S n' => S (plus n' m)
    end.
    
Fixpoint mult (n m : N) : N :=
  match n with
    | O => O
    | S n' => plus m (mult n' m)
    end.
    
Fixpoint minus (n m :N) : N :=
  match n,m with
    | O , _ => O
    | S _ , O => n
    | S n', S m' => minus n' m'
    end.
    
Fixpoint exp (base power : N) : N :=
  match power with
    | O => S O
    | S p => mult base (exp base p)
  end.

定理证明基本方法

化简（reflexivity）

	Theorem plus_0_n' : forall n:N , plus O n = n.
	Proof. intros n. reflexivity. Qed.

	Theorem plus_1_n' : forall n:N, plus (S O) n = S n.
	Proof. intros n. reflexivity. Qed.

	Theorem mult_0_n' : forall n:N, mult O n = O.
	Proof. intros n. reflexivity. Qed.

改写（rewrite）

	Theorem plus_id_example : forall n m : N,
		n = m -> (plus n n) = (plus m m).

	Proof.
	intros n m.
	intros H.
	rewrite -> H.
	reflexivity. 
	Qed.

证明过程说明

  • 将 n,m 添加到上下文，并记为n,m

  • 将蕴含的前提添加到上下文，并记为H

  • 使用H改写当前目标，采用从左向右改写，即将n替换成m

  • 化简

除了可以利用前提改写，还可以利用之前证明完成的“结论”进行改写。
(改写实际上是一种等价替换)

基于情况的分析

即分情况讨论。

	Theorem plus_1_neq_0 : forall n:N,
	neq (plus n (S O)) O = true.
	Proof.
	intros n. 
	destruct n as [| n'] eqn:E.
	-reflexivity.
	-reflexivity.
	Qed.

示例要证明的是 n + 1 ≠\neq= 0,使用了destruct关键字。
destruct 将n拆分成两种情况，第一种情况是定义中的'O',为不接收参数的构造器，第二种情况是定义中的'S ( n:N ) ' , 接收了一个参数 ，于是就有 ' as [ | n' ]' 对引入的变量进行命名；后面的eqn:E 对整个方程赋予了名称'E'.
