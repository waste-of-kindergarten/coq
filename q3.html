列表

数值序对（元组）

回顾在第一节中的更复杂的枚举构造——元组方法，利用之前定义的bit类型，定义了byte类型：

    Inductive byte : Type :=
        |bits (B0 B1 B2 B3 B4 B5 B5 B6 B7: bit).

在Inductive 类型定义中，每个构造子可以有任意多个参数。下面定义数值序对。

    Inductive natprod : Type :=
    | pair (n1 n2 : nat).

定义函数返回二元组的第一个分量和第二个分量。

    Definition fst (p: natprod) : nat :=
        match p with
            | pair x y => y
            end.

    Definition snd (p: natprod) : nat :=
        match p with
            | pair x y => y
        end.

定义函数交换二元组的两个元素。

    Definition swap_pair ( p : natprod) : natprod :=
        match p with
            | (x,y) => (y,x)
            end.

二元组可以用坐标的形式表示，《software foundation》中给的记法是

    Notation "( x , y )" := (pair x y).

当然，也可以先定义记法再定义相关函数。

数值列表

通过推广序对定义，定义数值列表为“一个列表要么是空的，要么是由一个数和另一个列表组成的序对。”

    Inductive natlist : Type :=
        | nil
        | cons (n :nat) (l : natlist).

《software foundation》对数值列表中定义了三个记法

    Notation "x :: l" := (cons x l) (at level 60,right associativity).

    Notation "[]" := nil.
    
    Notation "[ x ; .. ; y]" := (cons x .. (cons y nil) ..).

用这些记法构造列表：

    Definition mylist1 := 1 :: 2 :: 3 :: nil.

    Definition mylist2 := [1;2;3].

注意： 由于 ::定义为右结合，所以mylist1 定义等效于1 ::(2 :: (3 :: nil)).

常用列表操作

Repeat:

    Fixpoint repeat (n count : nat) : natlist :=
        match count with
            | 0 => nil
            | S count' => n :: (repeat n count')
            end.

构造一个列表，含有元素n，一共count个。

Length:

    Fixpoint length (l:natlist) : nat :=
        match l with
            | nil => 0
            | h ::t => S (length t)
            end.

计算列表的长度。

Append:

    Fixpoint append (l1 l2 : natlist) : natlist :=
        match l1 with
            | nil => l2
            | h :: t => h :: (append t l2)
            end.

Notation "x ++ y" := (append x y) (right associativity, at level 60).

将两个列表联系起来。

Head:

    Definition hd (default: nat) (l : natlist) : nat :=
        match l with
            | nil => default
            | h :: t => h
        end.

返回头元素，空列表返回默认元素(default).

Tail:

    Definition tl (l:natlist) : natlist :=
        match l with
            | nil => nil   
            | h :: t => t
            end.

返回列表尾（能看到除了头元素的列表是尾）。

列表的应用

通过列表，我们对之前证明方法进行复习。

使用归纳证明列表有关定理

  • 列表加法的结合律：
    Theorem app_assoc : forall l1 l2 l3 : natlist,
    (l1 ++ l2) ++ l3 = l1 ++ (l2 ++ l3).
    Proof.
    intros l1 l2 l3. 
    induction l1 as [| n l1' IHl1'].
    -reflexivity. (*空列表*)
    -simpl. rewrite -> IHl1'. reflexivity. 
    Qed.

induction 将列表进行归纳，
(1) 空列表， 直接可证（显然）.
(2) l1 = cons (n l1') , 假设 l1' 满足条件（设为IHl1'）,证明 l1 也满足.

  • 反转一个列表不改变其长度：

先定义反转函数

    Fixpoint rev (l:natlist) : natlist :=
        match l with
            | nil => nil   
            | h :: t => rev t ++ [h]
            end.

然后定义要证的定理

    Theorem rev_length : forall l : natlist,
    length (rev l) = length l.

先提出一个引理帮助证明

    Theorem app_length : forall l1 l2 : natlist,
    length (l1 ++ l2) = (length l1) + (length l2).
    Proof.
    intros l1 l2. induction l1 as [| n l1' IHl1'].
    - reflexivity.
    - simpl. rewrite -> IHl1'. reflexivity. 
    Qed.

这样利用归纳就可以对定理Theorem rev_length进行证明，证明过程如下：

    Proof.
    intros l. 
    induction l as [| n l' IHl1'].
    -reflexivity.
    -simpl. rewrite -> app_length.
    simpl. rewrite -> IHl1'. rewrite plus_comm. reflexivity.
    end.

Options 可选类型

如果你学过一些主流编程语言，那么你将对列表的引用很熟悉，列表的引用将列表中特定位置的元素返回。

我们以python语言举例，

    >>> ls = [1,2,3,4]
    >>> print(ls[0])
    1

当引用的数字超过列表时，将会报错，Coq中要求对每种情况必须有返回值，所以对于越界的情况，需要特殊的处理方式，一种方法是定义一种新的数据类型，包含了自然数（因为我们的里列表目前只能容纳自然数）和'无意义'.

    Inductive natoption : Type :=
        | Some (n: nat)
        | None.

目前可以定义一个良好性质的函数来返回列表特定位置的元素：

    Fixpoint nth (l:natlist) (n:nat) : natoption :=
        match l with
            | nil => None
            | a :: l' => match n with
                            | 0 => Some a
                            | S n' => nth l' n'
                            end
            end.

(*条件表达式版本*)
    Fixpoint nth' (l:natlist) (n:nat) :natoption :=
        match l with
            | nil => None
            | a :: l' => if n =? 0 then Some a else nth' l' (pred n)
            end.
